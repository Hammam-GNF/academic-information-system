# Academic Information System Use Case

This document describes the use cases of the Academic Information System.

The purpose of this document is to define system interactions between users and academic processes.

---

# Actors

Initial system actors:

## Administrator

Responsible for managing the academic system.

Main responsibilities:

* Manage users
* Manage academic structure
* Manage students
* Manage teachers
* Manage system configuration

---

## Teacher

Responsible for learning activities.

Main responsibilities:

* Manage teaching schedule
* Record attendance
* Manage assessments
* Monitor student progress

---

## Student

Responsible for accessing academic information.

Main responsibilities:

* View schedule
* View attendance
* View assessment results
* View academic reports

---

# Use Case Categories

## User Management

Future use cases:

* Create user
* Update user
* Assign role
* Manage permissions

---

## Academic Management

Future use cases:

* Manage academic year
* Manage semester
* Manage classes
* Manage subjects

---

## Learning Management

Future use cases:

* Create teaching schedule
* Manage learning sessions
* Record learning activities

---

## Attendance Management

Future use cases:

* Record attendance
* View attendance history
* Generate attendance report

---

## Assessment Management

Future use cases:

* Create assessment
* Input scores
* Calculate final score
* Generate grade report
